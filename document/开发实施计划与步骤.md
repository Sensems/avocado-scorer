# 极速记分小程序MVP：开发实施计划与步骤

基于产品需求文档（PRD）的大纲架构与前端初步规划，结合高级 UI/UX 设计规范（ui-ux-pro-max & frontend-design），特制定以下具备强执行力的底层开发实施计划。

本计划遵循 **Vue3 Composition API + TypeScript 强类型 + Pinia 集中状态管理** 的最佳实践，确保应用的高性能与高可维护性。

---

## 一、 技术栈与架构选型 (Tech Stack)

*   **核心框架**: `unibest` (uni-app + Vue3 + Vite + TypeScript) - 提供极速编译与优秀的工程化体验。
*   **状态管理**: `Pinia` - 聚焦核心记分状态流水线。
*   **UI/样式库**: 原生 CSS Variables (驱动全局 Dark Mode) + 深度定制的 `uView Pro` (处理 Toast/Modal 等基础交互)。
*   **绘图引擎**: `lime-painter` (JSON-driven 的 Canvas 生成器) - 用于多端兼容的高清海报渲染。
*   **工程化**: ESLint + Prettier + Husky (规范代码流)，强制遵循 TSLint 最佳实践。

---

## 二、 核心 UI/UX 体验规范落地策略

在编码实施中，以下设计规范必须作为高优验收标准：
1.  **极简生机风 (Dark Mode Base)**：
    *   底层背景严格采用 `#1E1E1E` 避免全黑的高对比度刺眼问题。
    *   主交互节点（卡片选中态、确认键）采用品牌色 **牛油果绿 (`#8A9A5B`)**，辅以顺滑的 Glow（发光）阴影特效体现高级感。
2.  **微交互与物理反馈体验 (Micro-Animations & Haptics)**：
    *   所有按键必须封装通过 `uni.vibrateShort({ type: 'light' })` 触发的 Taptic Engine 反馈。
    *   数字变化必须通过 `CSS Transform` 配合 `requestAnimationFrame` 实现滚字动效 (Number Scroller)。

---

## 三、 分阶段实施步骤 (Execution Steps)

### 阶段一：工程基建与响应式重型资产打造 (2天)
*目标：把重复轮子造好，类型接口定义清晰。*

1.  **工程骨架搭建**：
    *   基于 `unibest` 初始化，清理无用模板，配置别名 `@/` 指向。
    *   在 `App.vue` 注入全局 CSS 自定义变量。确保无路由白屏割裂感。
2.  **全局状态树 (Store) 定义**：
    *   设计 `useScoringStore.ts`。定义核心 `Player` 接口（ID, 姓名, 头像, 分数, 是否虚拟替身）及 `ScoreLog` 接口（流水账：from, to, amount, timestamp）。
    *   实现防篡改和带撤销堆栈的 Action 逻辑。
3.  **核心原子 UI 封装**：
    *   `UI-HapticBtn`: 震动反馈按键。
    *   `UI-Avatar`: 支持高亮发光态的玩家头像挂件。
    *   `UI-NumScroll`: 高性能数字滚动组件。

### 阶段二：对局大厅与“替身”入座流 (2天)
*目标：极致丝滑的开局体验，绝不让授权阻塞核心流程。*

1.  **大厅路由与 UI**：开发等待大厅页面，绘制高亮全屏邀请二维码。
2.  **扫码与参数解析**：拦截小程序 `onLaunch` 与路由参数，解析房间号并加入房间。
3.  **虚拟替身模块**：
    *   开发快捷输入弹窗。
    *   在 Store 中注入模拟玩家数据，保证房主能操作“替身”进行后续的所有收付款逻辑，解决线下凑角难题。

### 阶段三：2x2 沉浸式收银操作台（项目核心战役） (3天)
*目标：高频盲操零失误，渲染 60fps 稳定反馈。*

1.  **阵列试图与锚点引擎**：
    *   构建 2x2 网格，绑定选中状态 (`activeTargetId`) 逻辑。第一人称视觉高亮。
2.  **4x5 常驻收银键盘开发**：
    *   利用 Grid 布局实现底部吸附键盘，配置 `+10` / `+50` 快捷加码。
    *   编写核心收银结算函数（处理两方结算），编写 `🌊 全收` 方法遍历扣除其他三家底分。
3.  **防呆流水气泡与回滚**：
    *   监听 Store 流水栈顶元素，UI 浮出动画播报“A 收 B 50”。
    *   提供侧边回滚按键，实装 `undoLastAction` 逆向计算回退。

### 阶段四：称号引擎与情绪战报海报 (2天)
*目标：社交裂变催化剂，动态图文无缝生成。*

1.  **称号数据判定模块**：
    *   根据 PRD 附录实现纯函数的 `TitleEngine.ts`。入参：全局数据流，出参：玩家称号对象与特效等级。
2.  **海报渲染与分享**：
    *   运用 `lime-painter` 构建底层 JSON 模板（深色底、情绪背景、大字号得分）。
    *   调用 `wx.showShareImageMenu` 实现一键群嘲讽功能，附加上精心配置的缩略图 Title。

### 阶段五：兜底优化与发布态准备 (1天)
1.  **并发与容错测试**：针对高频点击按键加入节流，针对键盘撤销堆栈上下限加入防御性断言。
2.  **性能分析分包**：利用 Webpack/Vite analyzer 剥离海报渲染引擎到子包，缩减主包体积至 1.5MB 以下。
3.  **沉浸式适配**：全盘核对 iOS 底部小黑条 `safe-area-inset-bottom` 和刘海屏。

---

> 👨‍💻 **Architect Note**: 整体开发我们将采用极度内聚的组件设计。记分台的交互密度极高，需严格避免不必要的模板重渲染。数据流转务必保持单向，所有资金增减强制走 Store 的 Action 以备完善撤销（Undo）机制。
